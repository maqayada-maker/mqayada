# منصة مقايضة — سوق عكسي لمقايضة المديونية

**مقايضة** هي أول منصة سعودية متخصصة في «مقايضة» المديونية القائمة: بدلاً من أن يطرق العميل أبواب البنوك بحثاً عن إعادة تمويل أفضل، يقدّم طلباً واحداً مجهول الهوية، فيتنافس مستشارو التمويل المعتمدون على تقديم أفضل العروض له.

> مشاركة في **هاكاثون أمد 2026** — مسار الابتكار في التقنية المالية.

---

## المشكلة

- ملايين المقترضين في المملكة مرتبطون بتمويلات قديمة بنسب ربح أعلى من أسعار السوق الحالية.
- إعادة التمويل اليوم تتطلب زيارات متعددة للبنوك، ومشاركة البيانات الشخصية الكاملة مع كل جهة، وأسابيع من الانتظار — دون قدرة حقيقية على مقارنة العروض.

## الحل

سوق عكسي (Reverse Marketplace) يقلب المعادلة:

1. **طلب مجهول الهوية** — يُدخل العميل بياناته المالية فقط (الراتب، المديونية، المدة) دون أي بيانات تعريفية.
2. **مزايدة تنافسية** — يرى المستشارون المعتمدون الملف المالي ويتنافسون بعروض حقيقية.
3. **مقارنة ذكية** — يقارن العميل العروض جنباً إلى جنب مع تحليل ذكي يوصي بالأنسب.
4. **تواصل آمن** — بعد قبول العرض فقط تصل بيانات التواصل للمستشار الفائز.

## لماذا نحن مختلفون؟

| وجه المقارنة | منصة مقايضة | القنوات التقليدية |
|---|---|---|
| التخصص | مقايضة المديونية القائمة حصراً | تمويل جديد عام غير متخصص |
| آلية العمل | جهات التمويل تتنافس على العميل | العميل يطرق أبواب البنوك بنفسه |
| الخصوصية | هوية مخفية حتى قبول العرض | بيانات كاملة لدى كل جهة |
| الشفافية | مقارنة العروض جنباً إلى جنب | عروض متفرقة يصعب مقارنتها |
| السرعة | عروض تنافسية خلال 72 ساعة | أسابيع من المراجعات والانتظار |

**مثال محسوب:** مديونية 300,000 ريال بفارق 3% في نسبة الربح على 5 سنوات ≈ **45,000 ريال توفيراً للعميل**.

## أبرز المزايا المنفَّذة

- **أربع بوابات كاملة**: العميل، المستشار، المشرف، والإدارة — بصلاحيات وتدفقات مستقلة.
- **إخفاء هوية العميل** طوال مرحلة العروض مع فصل طبقة الهوية عن البيانات المالية.
- **مساعد مالي ذكي** (OpenAI) يحلّل العروض المقدّمة ويوصي بالأنسب مع شرح الأسباب.
- **حاسبة التوفير** التفاعلية على الصفحة الرئيسية: قارن نسبتك الحالية بالنسبة المتوقعة وشاهد التوفير الشهري والإجمالي.
- **لوحة متصدرين ونظام أداء** يحفّز المستشارين على تقديم أفضل العروض.
- **إشعارات فورية** (Web Push) وبريد إلكتروني (Resend) لتحديثات الطلبات والعروض.
- **إشراف إداري** على كل صفقة قبل تمرير بيانات التواصل.

## البنية التقنية

| الطبقة | التقنية |
|---|---|
| الواجهة | React 18 + Vite + TypeScript + Tailwind CSS (RTL بالكامل) |
| الخادم | Node.js + Express + TypeScript |
| قاعدة البيانات | PostgreSQL + Drizzle ORM |
| الذكاء الاصطناعي | OpenAI API (تحليل العروض والتوصيات) |
| البريد | Resend |
| الإشعارات | Web Push (VAPID) |

## التشغيل محلياً

</div>

```bash
pnpm install

# متغيرات البيئة المطلوبة:
# DATABASE_URL      (PostgreSQL)
# JWT_SECRET        (مفتاح توقيع الجلسات — أي نص عشوائي طويل)
# OPENAI_API_KEY    (المساعد الذكي)
# RESEND_API_KEY    (البريد — اختياري)
# VAPID_PUBLIC_KEY / VAPID_PRIVATE_KEY / VAPID_SUBJECT  (الإشعارات — اختياري)

# مزامنة مخطط قاعدة البيانات
pnpm --filter @workspace/db run push

# تشغيل الخادم والواجهة
pnpm --filter @workspace/api-server run dev
pnpm --filter @workspace/mqayada run dev
```
